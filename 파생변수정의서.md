## 2. 원천 데이터 정의

파생변수 산출에 사용되는 원천 데이터(Raw Data) 요약입니다. 상세 스키마는 스키마구조설계서를 참조합니다.

### 2.1 원천 테이블 및 주요 변수

| 원천 | 테이블 | 수집 API | 주요 원천변수 |
|------|--------|---------|-------------|
| KOBIS 일별 박스오피스 | `daily_box_office` | searchDailyBoxOfficeList | date, rank, movieNm, openDt, salesAmt, salesShare, salesAcc, audiCnt, audiAcc, scrnCnt, showCnt |
| KOBIS 주간 박스오피스 | `weekly_box_office` | searchWeeklyBoxOfficeList | showRange, weekGb, rank, movieNm, salesAmt, salesAcc, audiCnt, audiAcc, scrnCnt, showCnt |
| KOBIS 영화 상세 | `kobis_movie` | searchMovieInfo | movieCd, movieNm, openDt, genreAlt, showTm, directors, actors |
| TMDB 영화 | `movies` | /movie/top_rated | id, title, releaseDate, voteAverage, voteCount, popularity, genres |

### 2.2 원천변수 상세

| 변수명 | 테이블 | 타입 | 단위 | 설명 |
|--------|--------|------|------|------|
| `audiCnt` | daily_box_office, weekly_box_office | BIGINT | 명 | 당일/해당기간 관객수 |
| `audiAcc` | daily_box_office, weekly_box_office | BIGINT | 명 | 누적 관객수 |
| `salesAmt` | daily_box_office, weekly_box_office | BIGINT | 원(KRW) | 당일/해당기간 매출액 |
| `salesAcc` | daily_box_office, weekly_box_office | BIGINT | 원(KRW) | 누적 매출액 |
| `salesShare` | daily_box_office | DOUBLE | % | 매출 점유율 (KOBIS API 원천 제공) |
| `scrnCnt` | daily_box_office, weekly_box_office | BIGINT | 개 | 스크린 수 |
| `showCnt` | daily_box_office, weekly_box_office | BIGINT | 회 | 상영 횟수 |
| `openDt` | daily_box_office, weekly_box_office | VARCHAR | - | 개봉일 (YYYYMMDD 또는 YYYY-MM-DD) |
| `date` | daily_box_office | VARCHAR | - | 기준일자 (YYYY-MM-DD) |
| `showRange` | weekly_box_office | VARCHAR | - | 상영기간 (YYYYMMDD~YYYYMMDD) |
| `weekGb` | weekly_box_office | VARCHAR | - | 주간구분 (0=주간, 1=주말) |
| `movieNm` | daily_box_office, weekly_box_office | VARCHAR | - | 영화명 |
| `voteAverage` | movies | DOUBLE | 점(0~10) | TMDB 평균 평점 |
| `releaseDate` | movies | VARCHAR | - | 개봉일 (YYYY-MM-DD) |
| `genres` | movie_genres | VARCHAR | - | 장르명 (ElementCollection) |

## 3. 파생변수 상세 정의

### 3.1 일별 파생 통계 (DerivedStatsDto)

> **산출 위치**: `KobisService.getDerivedStats()`  
> **API 엔드포인트**: `GET /api/boxoffice/derived?date=YYYY-MM-DD`  
> **용도**: 특정 날짜의 Top 10 영화 간 효율 비교 분석

| # | 파생변수명 | 한글명 | 산출식 | 원천변수 | 결과 타입 | 단위 | 반올림 | 분모 0 처리 |
|---|-----------|--------|--------|---------|----------|------|--------|------------|
| 1 | `daysSinceRelease` | 개봉 경과일수 | `ChronoUnit.DAYS.between(openDt, targetDate)` | openDt, targetDate | int | 일 | - | - |
| 2 | `audiPerScreen` | 스크린당 관객수 | `audiCnt / scrnCnt` | audiCnt, scrnCnt | double | 명/스크린 | 소수점 1자리 | scrnCnt=0 → 0.0 |
| 3 | `audiPerShow` | 회차당 관객수 | `audiCnt / showCnt` | audiCnt, showCnt | double | 명/회 | 소수점 1자리 | showCnt=0 → 0.0 |
| 4 | `screenShare` | 스크린 점유율 | `(scrnCnt / totalScreens) × 100` | scrnCnt, 당일 전체 scrnCnt 합산 | double | % | 소수점 2자리 | totalScreens=0 → 0.0 |

**변수 간 관계도**

```
daily_box_office (특정 date 기준)
│
├─ audiCnt ──────────┬── audiCnt / scrnCnt ──→ [2] audiPerScreen
│                    └── audiCnt / showCnt ──→ [3] audiPerShow
├─ scrnCnt ──────────┬── scrnCnt / Σ(scrnCnt) × 100 ──→ [4] screenShare
│                    └── (audiPerScreen 분모)
├─ showCnt ──────────── (audiPerShow 분모)
├─ openDt  ──────────── DAYS.between(openDt, date) ──→ [1] daysSinceRelease
└─ salesShare ────────→ (원천 그대로 전달, 파생 아님)
```

**산출 프로세스**

```
1. daily_box_office 테이블에서 특정 date 기준 전체 레코드 조회 (Top 10)
2. totalScreens = Σ scrnCnt (조회된 전체 영화의 스크린 합계)
3. 각 영화별로:
   a. daysSinceRelease = DAYS.between(openDt, targetDate)
   b. audiPerScreen = audiCnt / scrnCnt  (scrnCnt > 0일 때)
   c. audiPerShow = audiCnt / showCnt    (showCnt > 0일 때)
   d. screenShare = scrnCnt / totalScreens × 100
4. 반올림 적용 후 DerivedStatsDto 생성
```

### 3.2 영화 추적 통계 (MovieTrackingDto)

> **산출 위치**: `KobisService.getMovieTracking()`  
> **API 엔드포인트**: `GET /api/boxoffice/tracking?movieNm=영화명`  
> **용도**: 개별 영화의 개봉 후 일별 성과 추이 분석

| # | 파생변수명 | 한글명 | 산출식 | 원천변수 | 결과 타입 | 단위 | 반올림 | 분모 0 처리 |
|---|-----------|--------|--------|---------|----------|------|--------|------------|
| 5 | `daysSinceRelease` | 개봉 경과일수 | `ChronoUnit.DAYS.between(openDt, date)` | openDt, date | int | 일 | - | - |
| 6 | `weekNumber` | 개봉 주차 | `(daysSinceRelease / 7) + 1` | daysSinceRelease | int | 주차 | 정수 나눗셈 | - |
| 7 | `audiPerScreen` | 스크린당 관객수 | `audiCnt / scrnCnt` | audiCnt, scrnCnt | double | 명/스크린 | 소수점 1자리 | scrnCnt=0 → 0.0 |
| 8 | `audiPerShow` | 회차당 관객수 | `audiCnt / showCnt` | audiCnt, showCnt | double | 명/회 | 소수점 1자리 | showCnt=0 → 0.0 |

**변수 간 관계도**

```
daily_box_office (특정 movieNm 기준, 날짜순 정렬)
│
├─ openDt + date ──→ DAYS.between() ──→ [5] daysSinceRelease
│                                           │
│                                           └── / 7 + 1 ──→ [6] weekNumber
├─ audiCnt / scrnCnt ──→ [7] audiPerScreen
└─ audiCnt / showCnt ──→ [8] audiPerShow
```

**산출 프로세스**

```
1. daily_box_office에서 movieNm 기준 날짜 오름차순 전체 레코드 조회
2. openDt = 첫 번째 레코드의 개봉일 (기준일)
3. 각 일별 레코드에 대해:
   a. daysSinceRelease = DAYS.between(openDt, 해당 date)
   b. weekNumber = daysSinceRelease / 7 + 1
   c. audiPerScreen = audiCnt / scrnCnt
   d. audiPerShow = audiCnt / showCnt
4. 반올림 적용 후 MovieTrackingDto 생성
```

**weekNumber 산출 예시**

| 개봉 경과일(daysSinceRelease) | 산출식 | weekNumber |
|------|------|------|
| 0일 (개봉 당일) | 0 / 7 + 1 | 1주차 |
| 6일 | 6 / 7 + 1 | 1주차 |
| 7일 | 7 / 7 + 1 | 2주차 |
| 13일 | 13 / 7 + 1 | 2주차 |
| 14일 | 14 / 7 + 1 | 3주차 |

### 3.3 주간 트렌드 집계 (WeeklyTrendDto)

> **산출 위치**: `KobisService.getWeeklyTrends()` → `aggregateByMonth()`, `aggregateBySeason()`  
> **API 엔드포인트**: `GET /api/boxoffice/weekly/trends`  
> **용도**: 월별/계절별 박스오피스 시장 규모 및 트렌드 분석

| # | 파생변수명 | 한글명 | 산출식 | 원천변수 | 결과 타입 | 단위 |
|---|-----------|--------|--------|---------|----------|------|
| 9 | `period` (월별) | 기간 (월) | `showRange[0:4] + "-" + showRange[4:6]` | showRange | String | "YYYY-MM" |
| 10 | `period` (계절) | 기간 (계절) | 월 기준 계절 분류 (하단 참조) | showRange | String | "YYYY 계절" |
| 11 | `totalSales` | 기간 총 매출 | `Σ salesAmt` | salesAmt (기간 내 전체) | long | 원(KRW) |
| 12 | `totalAudience` | 기간 총 관객수 | `Σ audiCnt` | audiCnt (기간 내 전체) | long | 명 |
| 13 | `movieCount` | 고유 영화 수 | `COUNT(DISTINCT movieNm)` | movieNm | int | 편 |
| 14 | `topMovie` | 최다 관객 영화 | `movieNm WHERE MAX(Σ audiCnt GROUP BY movieNm)` | movieNm, audiCnt | String | - |
| 15 | `avgScreens` | 평균 스크린 수 | `Σ scrnCnt / COUNT(entries)` | scrnCnt | long | 개 |

**계절 분류 기준 (#10)**

| 월 | 계절명 | period 값 예시 |
|----|--------|---------------|
| 3~5월 | 봄 | "2025 봄" |
| 6~8월 | 여름 | "2025 여름" |
| 9~11월 | 가을 | "2025 가을" |
| 12, 1~2월 | 겨울 | "2025 겨울" |

**변수 간 관계도**

```
weekly_box_office (weekGb="0" 주간 데이터만 대상)
│
├─ showRange ──→ 앞 6자리 추출 ──→ [9]  period (월별: "YYYY-MM")
│                                 ──→ [10] period (계절: "YYYY 계절")
│
├─ [그룹핑: period 기준]
│   │
│   ├─ salesAmt ──→ Σ(salesAmt) ──→ [11] totalSales
│   ├─ audiCnt  ──→ Σ(audiCnt)  ──→ [12] totalAudience
│   ├─ movieNm  ──→ COUNT(DISTINCT) ──→ [13] movieCount
│   ├─ movieNm + audiCnt ──→ GROUP BY movieNm → SUM(audiCnt) → MAX ──→ [14] topMovie
│   └─ scrnCnt  ──→ Σ(scrnCnt) / COUNT ──→ [15] avgScreens
```

**산출 프로세스**

```
1. weekly_box_office에서 weekGb="0" (주간 데이터)만 필터링
2. 각 레코드의 showRange에서 기간 키 추출:
   - 월별: showRange 앞 6자리 → "YYYY-MM" 변환
   - 계절별: 월 번호로 계절 분류 → "YYYY 계절" 변환
3. 기간 키로 그룹핑 후 집계:
   a. totalSales = SUM(salesAmt)
   b. totalAudience = SUM(audiCnt)
   c. movieCount = COUNT(DISTINCT movieNm)
   d. topMovie = movieNm별 SUM(audiCnt) 중 최대값의 영화명
   e. avgScreens = SUM(scrnCnt) / COUNT(레코드)
4. 기간 키 오름차순 정렬 (TreeMap)
```

### 3.4 역대 흥행 순위 (AllTimeRankingDto)

> **산출 위치**: `KobisService.getAllTimeRankings()`  
> **API 엔드포인트**: `GET /api/boxoffice/alltime?sortBy=audience|sales&limit=20`  
> **용도**: 수집 기간 내 역대 관객수/매출액 기준 영화 순위

| # | 파생변수명 | 한글명 | 산출식 | 원천변수 | 결과 타입 | 단위 |
|---|-----------|--------|--------|---------|----------|------|
| 16 | `maxAudiAcc` | 최대 누적 관객수 | `MAX(audiAcc) per movieNm` | audiAcc | long | 명 |
| 17 | `maxSalesAcc` | 최대 누적 매출액 | `MAX(salesAcc) per movieNm` | salesAcc | long | 원(KRW) |
| 18 | `rank` | 흥행 순위 | 정렬 후 1-based 순번 부여 | maxAudiAcc 또는 maxSalesAcc | int | 위 |

**변수 간 관계도**

```
daily_box_office (전체 데이터)
│
├─ [그룹핑: movieNm 기준]
│   │
│   ├─ audiAcc ──→ MAX(audiAcc) ──→ [16] maxAudiAcc ──┐
│   └─ salesAcc ──→ MAX(salesAcc) ──→ [17] maxSalesAcc ─┤
│                                                       │
│                                       sortBy 기준 정렬 │
│                                                       ▼
│                                       순번 부여 ──→ [18] rank
```

**산출 프로세스**

```
1. daily_box_office 전체 데이터 조회
2. movieNm 기준으로 그룹핑
3. 각 그룹에서:
   a. maxAudiAcc = MAX(audiAcc)
   b. maxSalesAcc = MAX(salesAcc)
   c. openDt = 첫 번째 레코드의 openDt
4. sortBy 파라미터에 따라 정렬:
   - "audience" → maxAudiAcc 내림차순
   - "sales" → maxSalesAcc 내림차순
5. limit 적용 후 1-based rank 부여
```

### 3.5 프론트엔드 파생 통계

> **산출 위치**: React 컴포넌트 내 클라이언트 사이드 계산  
> **용도**: TMDB 데이터 기반 장르별/연도별/전체 통계 시각화

| # | 파생변수명 | 한글명 | 산출식 | 원천변수 | 산출 파일 | 단위 |
|---|-----------|--------|--------|---------|----------|------|
| 19 | `avgRating` | 전체 평균 평점 | `Σ voteAverage / COUNT(movies)` | voteAverage | `StatBanner.tsx` | 점(0~10) |
| 20 | `rating` (장르별) | 장르별 평균 평점 | `Σ voteAverage / COUNT(해당 장르 영화)` | voteAverage, genres | `GenreChart.tsx` | 점(0~10) |
| 21 | `평균평점` (연도별) | 연도별 평균 평점 | `Σ voteAverage / COUNT(해당 연도 영화)` | voteAverage, releaseDate | `YearTrendChart.tsx` | 점(0~10) |

**변수 간 관계도**

```
movies (TMDB 데이터)
│
├─ voteAverage ──→ Σ / COUNT(전체) ──→ [19] avgRating (전체)
│
├─ voteAverage + genres
│   └─ 장르별 그룹핑 ──→ Σ / COUNT(장르별) ──→ [20] rating (장르별)
│
└─ voteAverage + releaseDate
    └─ 연도별 그룹핑 ──→ Σ / COUNT(연도별) ──→ [21] 평균평점 (연도별)
```

**산출 상세**

| # | 그룹핑 키 | 집계 대상 | 반올림 | 비고 |
|---|----------|----------|--------|------|
| 19 | 없음 (전체) | 전체 영화의 voteAverage | 소수점 1자리 (`toFixed(1)`) | StatBanner에 표시 |
| 20 | genres 배열의 각 장르명 | 해당 장르에 속하는 영화의 voteAverage | 소수점 2자리 (`toFixed(2)`) | 한 영화가 복수 장르에 중복 집계됨 |
| 21 | releaseDate 앞 4자리 (연도) | 해당 연도 영화의 voteAverage | 소수점 2자리 (`toFixed(2)`) | YearTrendChart에 표시 |

| 19 | avgRating | 전체 평균 평점 | TMDB 통계 | Σ voteAverage / COUNT | 점 | 프론트 |
| 20 | rating (장르별) | 장르별 평균 평점 | TMDB 통계 | Σ voteAverage / COUNT(장르) | 점 | 프론트 |

## 6. 파생변수 재사용 관계

동일한 산출 로직이 여러 DTO에서 반복 사용되는 변수를 식별합니다.

| 파생변수 로직 | 사용 DTO | 비고 |
|-------------|---------|------|
| `audiCnt / scrnCnt` (스크린당 관객수) | DerivedStatsDto (#2), MovieTrackingDto (#7) | 동일 로직, 독립 산출 |
| `audiCnt / showCnt` (회차당 관객수) | DerivedStatsDto (#3), MovieTrackingDto (#8) | 동일 로직, 독립 산출 |
| `DAYS.between(openDt, date)` (경과일수) | DerivedStatsDto (#1), MovieTrackingDto (#5) | 동일 유틸 메서드 `calcDaysSinceRelease()` 사용 |
| `Σ voteAverage / COUNT` (평균 평점) | StatBanner (#19), GenreChart (#20), YearTrendChart (#21) | 그룹핑 키만 상이 |

## 7. 데이터 품질 규칙

### 7.1 분모 0 방지

| 파생변수 | 분모 | 방어 로직 | 기본값 |
|---------|------|----------|--------|
| audiPerScreen | scrnCnt | `scrnCnt > 0 ? ... : 0` | 0.0 |
| audiPerShow | showCnt | `showCnt > 0 ? ... : 0` | 0.0 |
| screenShare | totalScreens | `totalScreens > 0 ? ... : 0` | 0.0 |
| avgScreens | entries.size() | `items.isEmpty() ? 0 : ...` | 0 |

### 7.2 반올림 규칙

| 파생변수 | 반올림 방식 | 정밀도 | 구현 |
|---------|-----------|--------|------|
| audiPerScreen | 반올림 | 소수점 1자리 | `Math.round(x * 10.0) / 10.0` |
| audiPerShow | 반올림 | 소수점 1자리 | `Math.round(x * 10.0) / 10.0` |
| screenShare | 반올림 | 소수점 2자리 | `Math.round(x * 100.0) / 100.0` |
| rating (장르/연도) | 반올림 | 소수점 2자리 | `parseFloat(x.toFixed(2))` |
| avgRating (전체) | 반올림 | 소수점 1자리 | `x.toFixed(1)` |

### 7.3 날짜 형식 정규화

| 입력 형식 | 변환 | 적용 대상 |
|----------|------|----------|
| `YYYYMMDD` (8자리) | → `YYYY-MM-DD` (하이픈 삽입) | openDt (KOBIS API 원천) |
| `YYYY-MM-DD` (이미 정규) | 그대로 사용 | date (내부 저장 형식) |

`calcDaysSinceRelease()` 메서드에서 자동 감지 후 변환:
```
openDt.length() == 8 && !openDt.contains("-")
  → substring(0,4) + "-" + substring(4,6) + "-" + substring(6,8)
```

## 8. 검토 의견 및 개선 권고

### 8.1 현재 설계의 장점

| 항목 | 설명 |
|------|------|
| 분모 0 방지 | 모든 나눗셈 파생변수에 사전 분모 체크 적용 |
| 반올림 일관성 | 백엔드 `Math.round()` 패턴으로 통일 |
| 원천/파생 분리 | Entity(원천) ↔ DTO(파생) 명확히 분리되어 데이터 계보 추적 용이 |
| 실시간 산출 | DB에 파생값을 저장하지 않아 원천 데이터 변경 시 정합성 유지 |
| 유틸 공유 | `calcDaysSinceRelease()` 메서드를 DerivedStats와 MovieTracking이 공유 |

### 8.2 개선 권고 사항

| 우선순위 | 항목 | 현재 상태 | 권고 사항 | 사유 |
|---------|------|----------|----------|------|
| 높음 | screenShare 모수 | Top 10 영화의 scrnCnt 합산 | 변수명을 `screenShareInTop10`으로 변경하거나, 설명에 "Top 10 내 비중"임을 명시 | 전국 총 스크린 대비 점유율로 오해 가능 |
| 높음 | getAllTimeRankings 성능 | `findAll()` 후 인메모리 그룹핑 | 데이터 누적 시 JPQL `GROUP BY movieNm` + `MAX()` 집계 쿼리로 전환 | 1년치 데이터 기준 약 3,650건 전체 로드 |
| 중간 | avgScreens 정의 | `Σ scrnCnt / COUNT(entries)` | "엔트리당 평균 스크린"으로 정의를 명확히 하거나, `영화당 평균 = Σ scrnCnt / COUNT(DISTINCT movieNm)`으로 변경 | 같은 영화가 여러 주에 걸쳐 집계될 때 왜곡 가능 |
| 중간 | 프론트엔드 파생변수 | 클라이언트 사이드 계산 | 데이터량 증가 시 서버 사이드 API로 전환 검토 | 현재 200편 이하이므로 당장은 문제 없음 |
| 낮음 | openDt 형식 | YYYYMMDD / YYYY-MM-DD 혼용 | ETL 적재 시점에 YYYY-MM-DD로 통일 정규화 | `calcDaysSinceRelease()`에서 런타임 감지하는 방어 코드 의존 중 |
| 낮음 | weekNumber 경계 | 개봉일 당일(day=0)이 1주차 | 의도된 설계라면 문서에 명시, 아니라면 `Math.ceil((days+1)/7.0)`으로 변경 검토 | 0~6일=1주차, 7~13일=2주차로 7일 단위 |
