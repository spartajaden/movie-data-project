## 3. 테이블 상세 정의

### 3.1 movies (TMDB 영화)

| 컬럼명         | 타입         | 제약조건 | 설명                                            |
| -------------- | ------------ | -------- | ----------------------------------------------- |
| id             | BIGINT       | PK       | TMDB 영화 ID (자동생성 아님, TMDB 원본 ID 사용) |
| title          | VARCHAR(255) | NOT NULL | 한국어 영화 제목                                |
| original_title | VARCHAR(255) |          | 원어 제목                                       |
| overview       | TEXT         |          | 줄거리                                          |
| release_date   | VARCHAR(255) |          | 개봉일 (YYYY-MM-DD)                             |
| vote_average   | DOUBLE       |          | 평균 평점 (0~10)                                |
| vote_count     | BIGINT       |          | 투표 수                                         |
| popularity     | DOUBLE       |          | TMDB 인기도 지수                                |
| poster_path    | VARCHAR(255) |          | 포스터 이미지 경로                              |
| backdrop_path  | VARCHAR(255) |          | 배경 이미지 경로                                |
| movie_cd       | VARCHAR(255) |          | KOBIS 영화코드 (연동용)                         |

**특이사항**

- PK가 AUTO_INCREMENT가 아닌 TMDB 원본 ID를 직접 사용
- movie_cd를 통해 kobis_movie와 논리적으로 연결

---

### 3.2 movie_genres (TMDB 영화 장르 — 컬렉션 테이블)

| 컬럼명   | 타입         | 제약조건        | 설명                      |
| -------- | ------------ | --------------- | ------------------------- |
| movie_id | BIGINT       | FK → movies(id) | 영화 ID                   |
| genre    | VARCHAR(255) |                 | 장르명 (예: 액션, 드라마) |

**특이사항**

- JPA `@ElementCollection`으로 자동 생성되는 컬렉션 테이블
- movies와 1:N 관계 (한 영화에 여러 장르)
- FetchType.EAGER로 즉시 로딩

## 4. 테이블 간 관계

### 4.1 물리적 관계 (FK)

```
movies  ──── 1:N ────  movie_genres
  (id)                   (movie_id)
```

JPA `@ElementCollection` + `@CollectionTable`에 의해 자동 생성되는 유일한 물리적 FK 관계.

### 4.2 논리적 관계 (FK 제약 없음)

```
movies.movie_cd  ·· ·· ··  kobis_movie.movie_cd
  TMDB 영화에 KOBIS 영화코드를 매핑하여 두 데이터 소스를 연결

weekly_box_office.movie_cd  ·· ·· ··  kobis_movie.movie_cd
  주간 박스오피스 영화를 KOBIS 영화 상세와 연결

daily_box_office.movie_nm  ·· ·· ··  weekly_box_office.movie_nm
  동일 영화를 영화명 기준으로 일별↔주간 데이터 연결
```

**FK 제약을 두지 않은 이유**

- 외부 API에서 수집하는 데이터이므로 수집 순서·범위가 보장되지 않음
- 일별 박스오피스에는 movie_cd가 없고 movie_nm만 존재 (KOBIS API 일별 응답 구조)
- 데이터 수집 실패 시 FK 제약이 전체 저장을 차단하는 것을 방지

## 5. 인덱스 설계

| 테이블            | 인덱스                 | 컬럼                  | 용도                                   |
| ----------------- | ---------------------- | --------------------- | -------------------------------------- |
| daily_box_office  | IDX_date               | `date`                | 날짜별 박스오피스 조회, 30일 초과 삭제 |
| weekly_box_office | IDX_show_range_week_gb | `show_range, week_gb` | 기간+구분별 조회                       |
| kobis_code        | IDX_com_code           | `com_code`            | 공통코드 분류별 조회                   |

**현재 인덱스가 없지만 조회에 사용되는 컬럼** (추가 권장)

| 테이블           | 권장 인덱스      | 컬럼           | 사유                                    |
| ---------------- | ---------------- | -------------- | --------------------------------------- |
| daily_box_office | IDX_movie_nm     | `movie_nm`     | findByMovieNmOrderByDateAsc (추적 기능) |
| movies           | IDX_release_date | `release_date` | SUBSTRING 연도 범위 조회                |
| movies           | IDX_vote_average | `vote_average` | 평점순 정렬 조회                        |

## 6. 주요 조회 패턴

### 6.1 기본 조회

| 패턴             | 테이블                   | 쿼리 방식                       | 사용처                  |
| ---------------- | ------------------------ | ------------------------------- | ----------------------- |
| 평점순 영화 목록 | movies + movie_genres    | findAll(Sort)                   | TMDB 대시보드           |
| 연도 범위 필터링 | movies                   | JPQL SUBSTRING                  | 필터바 연도 범위        |
| 장르별 통계      | movies JOIN movie_genres | JPQL GROUP BY                   | 장르 차트               |
| 영화 검색        | movies                   | findByTitleContainingIgnoreCase | 검색 탭                 |
| 일별 박스오피스  | daily_box_office         | findByDateOrderByRankAsc        | KOFIC 순위/매출/관객 탭 |
| 날짜 목록        | daily_box_office         | DISTINCT date DESC              | 날짜 탐색기             |
| 주간 박스오피스  | weekly_box_office        | findByShowRange+WeekGb          | KOFIC 주간 탭           |
| 기간 목록        | weekly_box_office        | DISTINCT showRange              | 주간 탐색기             |
| 배틀 풀          | daily_box_office         | findAll → 영화별 MAX(audiAcc)   | 배틀 게임               |
| 30일 초과 삭제   | daily_box_office         | DELETE WHERE date < cutoff      | 자동 정리               |
| 공통코드 조회    | kobis_code               | findByComCode                   | 코드 조회 API           |

### 6.2 파생통계 탭 (DerivedStats)

| 조회 패턴       | 원본 테이블      | 파생 계산                                   | 출력 시각화                   |
| --------------- | ---------------- | ------------------------------------------- | ----------------------------- |
| 스크린당 관객수 | daily_box_office | audiCnt / scrnCnt                           | 수평 바 차트 (효율 순 정렬)   |
| 회차당 관객수   | daily_box_office | audiCnt / showCnt                           | 수평 바 차트 (집객력 순 정렬) |
| 스크린 점유율   | daily_box_office | 해당 영화 scrnCnt / 전체 scrnCnt 합계 × 100 | 파이 차트                     |
| 종합 파생 변수  | daily_box_office | 위 지표 + 경과일, 매출 점유율 결합          | 종합 테이블                   |

**파생 DTO 필드 (DerivedStatsDto)**

| 필드             | 계산식                            | 설명              |
| ---------------- | --------------------------------- | ----------------- |
| audiPerScreen    | audiCnt / scrnCnt                 | 스크린당 관객수   |
| audiPerShow      | audiCnt / showCnt                 | 상영회차당 관객수 |
| screenShare      | (scrnCnt / 전체 scrnCnt 합) × 100 | 스크린 점유율 (%) |
| salesShare       | 원본 sales_share 컬럼             | 매출 점유율 (%)   |
| daysSinceRelease | 기준일 - openDt                   | 개봉 후 경과일 수 |

### 6.3 추적 탭 (MovieTracking)

| 조회 패턴            | 원본 테이블                    | 쿼리 방식                   | 출력 시각화              |
| -------------------- | ------------------------------ | --------------------------- | ------------------------ |
| 추적 가능 영화 목록  | daily_box_office               | DISTINCT movieNm            | 영화 선택 칩 목록        |
| 영화별 일별 추이     | daily_box_office               | findByMovieNmOrderByDateAsc | 일별 상세 테이블         |
| 누적 관객수 추이     | daily_box_office → 주차별 집계 | 주차별 MAX(audiAcc)         | 라인 차트 (X: 개봉 주차) |
| 주차별 관객수        | daily_box_office → 주차별 집계 | 주차별 SUM(audiCnt)         | 바 차트                  |
| 스크린당 관객수 추이 | daily_box_office → 주차별 집계 | 주차별 AVG(audiPerScreen)   | 라인 차트                |

**파생 DTO 필드 (MovieTrackingDto)**

| 필드             | 계산식                     | 설명                 |
| ---------------- | -------------------------- | -------------------- |
| daysSinceRelease | date - openDt              | 개봉 후 경과일 수    |
| weekNumber       | (daysSinceRelease / 7) + 1 | 개봉 주차 번호       |
| audiPerScreen    | audiCnt / scrnCnt          | 당일 스크린당 관객수 |
| audiPerShow      | audiCnt / showCnt          | 당일 회차당 관객수   |

**프론트엔드 추가 집계 (주차별)**

| 집계               | 계산식                            | 설명                       |
| ------------------ | --------------------------------- | -------------------------- |
| 주간 관객수        | SUM(audiCnt) per weekNumber       | 해당 주차 총 관객수        |
| 누적 관객수        | MAX(audiAcc) per weekNumber       | 해당 주차 말 누적 관객수   |
| 평균 스크린당 관객 | AVG(audiPerScreen) per weekNumber | 해당 주차 평균 스크린 효율 |

### 6.4 역대순위 탭 (AllTimeRanking)

| 조회 패턴          | 원본 테이블      | 쿼리 방식                        | 출력 시각화                |
| ------------------ | ---------------- | -------------------------------- | -------------------------- |
| 역대 관객순 랭킹   | daily_box_office | 영화별 MAX(audiAcc) → 정렬       | 수평 바 차트 + 상세 테이블 |
| 역대 매출순 랭킹   | daily_box_office | 영화별 MAX(salesAcc) → 정렬      | 수평 바 차트 + 상세 테이블 |
| 관객수/매출액 토글 | —                | sortBy 파라미터 (audience/sales) | 토글 버튼으로 전환         |

**파생 DTO 필드 (AllTimeRankingDto)**

| 필드        | 계산식                    | 설명                         |
| ----------- | ------------------------- | ---------------------------- |
| rank        | 정렬 후 순번 부여         | 순위 (관객수 또는 매출 기준) |
| maxAudiAcc  | MAX(audiAcc) per movieNm  | 해당 영화의 최대 누적 관객수 |
| maxSalesAcc | MAX(salesAcc) per movieNm | 해당 영화의 최대 누적 매출액 |

## 9. 스키마 설계 검토 의견

### 9.1 현재 설계의 장점

| 항목               | 설명                                                 |
| ------------------ | ---------------------------------------------------- |
| 단순한 구조        | 테이블 간 복잡한 FK 없이 독립적으로 데이터 수집 가능 |
| 유연한 수집        | 외부 API 수집 순서에 구애받지 않음                   |
| 명확한 도메인 분리 | TMDB/KOFIC 데이터가 테이블 단위로 분리               |
| 장르 컬렉션        | @ElementCollection으로 다대다 관계를 간결하게 처리   |

### 9.2 개선 권장 사항

| 우선순위 | 항목                        | 현재             | 권장                                  | 사유                                         |
| -------- | --------------------------- | ---------------- | ------------------------------------- | -------------------------------------------- |
| 높음     | 날짜 타입                   | VARCHAR          | DATE / LocalDate                      | 날짜 비교·정렬 시 문자열보다 정확하고 효율적 |
| 높음     | daily_box_office 중복 방지  | 없음             | UNIQUE(date, movie_nm)                | 같은 날 같은 영화가 중복 저장될 수 있음      |
| 높음     | weekly_box_office 중복 방지 | 없음             | UNIQUE(show_range, week_gb, movie_cd) | 같은 기간 같은 영화 중복 가능                |
| 중간     | movie_nm 인덱스             | 없음             | IDX(movie_nm) on daily_box_office     | 영화 추적·배틀 풀 조회 성능 개선             |
| 중간     | 금액 단위 명시              | 없음             | 컬럼 코멘트 추가                      | sales_amt 등의 단위(원)를 DB 레벨에서 명시   |
| 낮음     | kobis_movie 다대다 정규화   | 쉼표 구분 문자열 | directors/actors 별도 테이블          | 감독·배우 기준 검색이 필요할 경우에만        |
| 낮음     | 소프트 딜리트               | 물리 삭제        | deleted_at 컬럼                       | 데이터 복구 필요 시 고려                     |
