# CineStats 프로젝트 개요서

## 1. 프로젝트 개요

| 항목       | 내용                                                                                                                                           |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| 프로젝트명 | CineStats — 영화 데이터 분석 대시보드                                                                                                          |
| 목적       | TMDB(글로벌 영화 데이터)와 KOFIC(한국 박스오피스 데이터)를 통합하여 영화 트렌드 분석, 시각화, AI 해설, 배틀 게임 등을 제공하는 웹 애플리케이션 |
| 개발 인원  | 3인 (mainug 팀)                                                                                                                                |
| 저장소     | GitHub — mainug/data-miniproject                                                                                                               |

## 3. 주요 기능

### 3.1 TMDB 영화 대시보드 (`/dashboard` — TMDB 탭)

| 탭     | 기능                                             |
| ------ | ------------------------------------------------ |
| 랭킹   | 평점·인기도·개봉일 기준 Top N 영화 조회          |
| 장르   | 장르별 영화 수·평균 평점 막대 차트               |
| 트렌드 | 연도별 개봉 수 + 평균 평점 이중축 차트           |
| 분석   | 평점 vs 인기도 산점도, Top N 비교 수평 막대 차트 |
| 검색   | 키워드 기반 영화 검색                            |

- **필터 바**: 연도 범위, 장르 다중선택, 정렬 기준, Top N 조절

### 3.2 KOFIC 박스오피스 대시보드 (`/dashboard` — KOFIC 탭)

| 탭        | 기능                                              |
| --------- | ------------------------------------------------- |
| 일별      | 날짜 선택 → 일별 박스오피스 순위·매출·관객 수     |
| 주간/주말 | 기간 선택 → 주간 또는 주말 박스오피스             |
| 트렌드    | 월별·계절별 박스오피스 추이 분석                  |
| 파생통계  | 스크린당 관객 수, 상영회당 관객 수, 스크린 점유율 |
| 추적      | 특정 영화의 주차별 누적 관객·효율 추이            |
| 역대순위  | 역대 관객 수·매출액 Top 20                        |

- **AI 해설**: 각 탭에 Ollama 기반 자연어 분석 코멘터리 제공

### 3.3 영화 상세 페이지 (`/movie/:id`)

- 포스터, 배경, 한글·원제, 평점, 투표 수, 인기도
- KOBIS 박스오피스 데이터 (누적 관객, 매출, 최대 스크린 수)
- 레이더 차트 (다차원 비교), 장르 내 순위 차트, 연도 동기 비교
- 장르 기반 유사 영화 추천 (최대 6편)

### 3.4 영화 배틀 게임 (`/battle`)

- 무작위 2편 매칭 → 관객 수·매출·스크린 수 중 랜덤 기준 비교
- 정답/오답 시각 피드백 (초록/빨강 플래시)
- 연승 기록 추적 (현재 연승 & 최고 연승)

### 3.5 게시판 관리 (`/board`)

- 영화 목록 조회 및 키워드 검색
- 영화 상세 조회 / 추가 / 수정 / 삭제 (CRUD)
- 중복 제출 방지 인터셉터 적용

## 5. 데이터베이스 스키마

### 5.1 테이블 구조

| 테이블              | 설명                                           | PK           |
| ------------------- | ---------------------------------------------- | ------------ |
| `movies`            | TMDB 영화 정보 (제목, 평점, 인기도, 포스터 등) | id (TMDB ID) |
| `movie_genres`      | 영화-장르 매핑 (ElementCollection)             | FK: movie_id |
| `daily_box_office`  | KOFIC 일별 박스오피스 (매출, 관객, 스크린 등)  | id (auto)    |
| `weekly_box_office` | KOFIC 주간/주말 박스오피스                     | id (auto)    |
| `kobis_movie`       | KOFIC 영화 마스터 (영화명, 장르, 감독, 배우)   | movie_cd     |
| `kobis_code`        | KOFIC 공통 코드 (장르코드 등)                  | id (auto)    |

### 5.2 주요 관계

```
movies ──(1:N)── movie_genres          [물리적 FK]
movies ··(1:1)·· kobis_movie           [논리적 연결: movie_cd]
daily_box_office ··(N:1)·· kobis_movie [논리적 연결: movie_nm]
weekly_box_office ··(N:1)·· kobis_movie[논리적 연결: movie_cd]
```

### 5.3 데이터 보존 정책

- 일별 데이터: 30일 보존 (자동 삭제)
- 주간 데이터: 365일 보존

## 6. REST API 엔드포인트

### 6.1 TMDB 영화 (`/api`)

| Method | URI                     | 설명                                     |
| ------ | ----------------------- | ---------------------------------------- |
| GET    | `/movies`               | 전체 영화 조회 (DB 비어있으면 자동 수집) |
| POST   | `/movies/refresh`       | 강제 새로고침 (삭제 후 재수집)           |
| GET    | `/movies/{id}`          | 영화 상세 (DB → TMDB API 폴백)           |
| GET    | `/movies/search?query=` | 영화 검색                                |

### 6.2 박스오피스 (`/api/boxoffice`)

| Method | URI                                 | 설명                 |
| ------ | ----------------------------------- | -------------------- |
| GET    | `/boxoffice?date=`                  | 일별 박스오피스      |
| GET    | `/boxoffice/dates`                  | 수집된 날짜 목록     |
| GET    | `/boxoffice/weekly?range=&weekGb=`  | 주간/주말 박스오피스 |
| GET    | `/boxoffice/weekly/ranges?weekGb=`  | 수집된 기간 목록     |
| GET    | `/boxoffice/weekly/trends`          | 월별·계절별 트렌드   |
| GET    | `/boxoffice/derived?date=`          | 파생 통계            |
| GET    | `/boxoffice/tracking?movieNm=`      | 영화별 주차 추적     |
| GET    | `/boxoffice/alltime?sortBy=&limit=` | 역대 순위            |

### 6.3 AI (`/api/ai`)

| Method | URI           | 설명                  |
| ------ | ------------- | --------------------- |
| POST   | `/commentary` | AI 분석 코멘터리 생성 |

### 6.4 배틀 (`/api/battle`)

| Method | URI            | 설명                |
| ------ | -------------- | ------------------- |
| GET    | `/battle/pool` | 배틀용 영화 풀 조회 |

## 9. 프로젝트 특징

1. **이중 데이터 소스 통합** — TMDB(글로벌)와 KOFIC(한국) 데이터를 하나의 대시보드에서 통합 분석
2. **풍부한 시각화** — 10종 이상의 Recharts 차트 (막대, 선, 산점도, 파이, 레이더 등)
3. **AI 기반 해설** — 로컬 LLM을 활용한 자연어 박스오피스 분석 코멘터리
4. **게이미피케이션** — 영화 배틀 퀴즈를 통한 사용자 참여 유도
5. **자동 데이터 수집** — 서버 기동 시 자동 백필, 보존 정책에 따른 자동 정리
6. **점진적 성능 저하(Graceful Degradation)** — Ollama 미가동 시 AI 기능만 비활성화, 나머지 정상 작동
7. **타입 안전성** — 프론트엔드 전체 TypeScript 적용
8. **다크/라이트 모드** — 시스템 설정 감지 + 사용자 선택 유지
